# openpyxl_rust v0.2 Design

## Goal

Four features to move openpyxl_rust from "basic writer" toward a real openpyxl drop-in:
1. Move cell storage to Rust (8-15x speedup target)
2. Alignment, Border, PatternFill styling
3. datetime/date support
4. load_workbook via calamine (data-only reading)

## Feature 1: Rust Cell Storage

### Problem
v0.1 stores cells in Python dicts, serializes to Rust at save(). The Python loop is the bottleneck — same speed as openpyxl for data collection, only the save phase is faster.

### Solution
Expose a `RustWorkbook` PyO3 class that stores cells in Rust memory. Python classes become thin wrappers.

**Rust side (PyO3 classes):**
- `RustWorkbook` — owns worksheets, provides save/save_to_buffer
- Methods: `add_worksheet(title)`, `set_cell(sheet, row, col, value)`, `set_cell_format(sheet, row, col, format_dict)`, `set_column_width(sheet, col, width)`, `set_row_height(sheet, row, height)`, `set_freeze_panes(sheet, row, col)`, `merge_range(sheet, r1, c1, r2, c2)`, `save(path)`, `save_to_buffer()`

**Python side changes:**
- `Workbook.__init__` creates a `RustWorkbook` internally
- `Worksheet.cell()` / `__setitem__` calls `rust_wb.set_cell()` immediately
- Setting `.font`, `.alignment`, `.border`, `.fill`, `.number_format` on a Cell calls `rust_wb.set_cell_format()`
- `wb.save()` calls `rust_wb.save()` directly — no serialization step

**Cell class redesign:**
- Cell becomes a lightweight proxy: stores (sheet_idx, row, col) + reference to RustWorkbook
- Getting `.value` reads from Rust; setting `.value` writes to Rust
- Format attributes (font, alignment, etc.) are cached Python-side and flushed to Rust on change

## Feature 2: Alignment, Border, PatternFill

### Python classes

```python
class Alignment:
    def __init__(self, horizontal=None, vertical=None, wrap_text=False,
                 shrink_to_fit=False, indent=0, text_rotation=0):

class Side:
    def __init__(self, style=None, color=None):

class Border:
    def __init__(self, left=Side(), right=Side(), top=Side(), bottom=Side()):

class PatternFill:
    def __init__(self, fill_type=None, start_color=None, end_color=None):
```

### Rust mapping (rust_xlsxwriter::Format)
- Alignment.horizontal → `format.set_align(FormatAlign::Center)` etc.
- Alignment.vertical → `format.set_align(FormatAlign::VerticalCenter)` etc.
- Alignment.wrap_text → `format.set_text_wrap()`
- Side.style → `FormatBorder::Thin`, `FormatBorder::Medium`, etc.
- Side.color → `format.set_border_left_color()` etc.
- PatternFill.fill_type → `format.set_pattern(FormatPattern::Solid)` etc.
- PatternFill.start_color → `format.set_background_color()`

### Cell attributes
Cell gets: `.alignment`, `.border`, `.fill` (all None by default, alongside existing `.font`, `.number_format`)

## Feature 3: datetime Support

### Behavior (matching openpyxl)
- Python `datetime` → Excel serial float + auto-apply "yyyy-mm-dd hh:mm:ss" number format
- Python `date` → Excel serial float + auto-apply "yyyy-mm-dd" number format
- Conversion: days since 1899-12-30 (Excel epoch), with the Lotus 1-2-3 bug (day 60 = Feb 29 1900)

### Implementation
- Rust side: detect Python datetime/date via PyO3, convert to f64 serial number, write with format
- rust_xlsxwriter has ExcelDateTime but we can just write the f64 + number format directly

## Feature 4: load_workbook via calamine

### Scope
Data-only reading. calamine cannot read styles/formatting — this is documented and acceptable.

### API
```python
from openpyxl_rust import load_workbook

wb = load_workbook("input.xlsx", data_only=True)
ws = wb.active
ws['A1'].value  # cell value (str, float, int, bool, datetime, or None)
wb.sheetnames   # list of sheet names
```

### Implementation
- Add `calamine` crate to Cargo.toml
- Rust `_load_workbook(path)` function returns a Python dict: {sheet_names: [...], sheets: {name: [[cell_values...], ...]}}
- Python `load_workbook()` constructs a Workbook/Worksheet/Cell tree from the dict
- Loaded workbooks are mutable — users can modify and save

### Limitations (documented)
- No style/formatting read-back
- Formulas return cached values (data_only behavior)
- Merged cell ranges not preserved on read

## Dependency Changes
- Add `calamine = "0.33"` to Cargo.toml

## Test Plan

- Compat tests: write with styles, read back with openpyxl, verify formatting
- datetime compat tests: write dates, read back, verify values
- load_workbook tests: write with openpyxl, read with load_workbook, verify values
- Updated benchmark comparing v0.2 (Rust storage) vs openpyxl
